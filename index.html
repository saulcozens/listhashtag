Twitter List HashTag Search Service

Checking for hashtags on your twitter lists

Have you ever wanted to know whether someone on one of your Twitter lists has used a specific hashtag recently? No? Well I did. So I created this webservice to tell me.

I use this service to drive Arduino projects that trigger an event (like a door opening, a light flashing or a siren going off) whenever someone I know uses a given hashtag.

For example, my friend @yorkhannah want to train her colleague to refrain from making innappropriate comments in the office. I have linked an Arduino to an air freshener so that it lets off a cloud of pungent pine perfume when she (or one of her list of allowed activators) tweet the phrase #racismstinks.

I created the service to be reasonbly generic, so it will take any twitter screename/list combination on the url along with a comma separated list of hashtags and return a simple true/false result. The service also returns the id of the last tweet checked so that the calling Arduino (or other app) can keep track of what tweets have already been searched. This id can be include on the request url next time to prevent repeated checking the same tweets.

The url format is: http://listhashtag.appspot.com/screen_name/list_name/hashtag[,hashtag]/[since_id]
for example: http://listhashtag.appspot.com/yorkhannah/badracistlist/badracist,racismstinks for the first call, which might return
[True,'200593656258887680'] as a result
then you would call http://listhashtag.appspot.com/yorkhannah/badracistlist/badracist,racismstinks/200593656258887680 at some point later

Putting all of the Twitter API stuff in this webservice makes it really easy for a simple app (like an Arduino program) to do something that might otherwise require alsorts of messing about with OAuth and SSL libraries and such. Now it's just a http GET. Pretty natty huh?

Rate limits

There is a minor problem though. Twitter rate limits all anonymous API requests by the source IP address and as this service is running on the Google App Engine it shares the rate limit with all other Google App Engine apps. So you are welcome to use this service anonymously, but don't be surprised when you get API rate limit errors.

If you want to use this service in anger, you should authorise it to use your Twitter account. It will then be able to use YOUR API allocation for any requests to your lists. It only gets permission to read your status feed so it can't publish tweets on your behalf (I hate apps that do that, don't you?). But still be aware, it will use some of the same allocation that your normal Twitter clients (Tweetdeck, Tweetbot, etc.) use, so if you have a heavy load application you might want to create a separate Twitter account and authorise this service to use that instead.
